# ⚡ PowerNode

**Self-hosted, multi-node control panel for game servers, bots and generic apps.**

Одна панель-мастер + произвольное число нод с демоном (`wingsd`), общающихся по
защищённому токен-протоколу. Полный RBAC, 2FA на каждый вход, живая консоль,
которая реально переподключается, и модульная система деплоя ("eggs") —
Minecraft, Discord/Telegram-боты, Python-приложения или любой свой Docker-образ.

## ⚡ Установка в одну команду

```bash
bash <(curl -sSL https://raw.githubusercontent.com/superbodik/PowerNode/main/install.sh)
```

Ubuntu 22.04/24.04, от root. Установщик сам спросит язык (English/Русский) и
предложит интерактивное меню (`whiptail`, с текстовым fallback):

1. **Мастер-панель** (Backend + Frontend + PostgreSQL + Redis)
2. **Демон-нода** (`wingsd` + Docker)
3. **Всё сразу** — на одном сервере
4. Удалить (данные серверов и БД сохраняются)
5. **Полное удаление** — сносит БД, Docker и все данные; требует ввести
   `DELETE` заглавными буквами для подтверждения

При установке панели установщик сам спрашивает домен (и выпускает HTTPS через
certbot/Let's Encrypt) и заводит первого админ-аккаунта — регистрации через UI
нет и не будет, только `panel-admin`. При установке ноды — просит
`daemon_token`, который панель показывает один раз на странице **Nodes → Add
node**.

Обновление существующей установки — та же команда, с флагом:

```bash
PANEL_UPDATE=1 bash <(curl -sSL https://raw.githubusercontent.com/superbodik/PowerNode/main/install.sh)
```

Версии — по-настоящему semver: `VERSION` в корне репозитория, релизы —
осознанные теги (`git tag vX.Y.Z`), а не "update available" на каждый коммит.

## 📦 Что внутри

| | |
|---|---|
| 🔐 **RBAC + subusers** | Роли/права + отдельный уровень: расшарить конкретный сервер другому пользователю с точечным набором прав, не выдавая роль на всю панель |
| 🛡️ **2FA везде** | TOTP с настоящим QR-кодом, отдельный экран верификации (не поле снизу формы), rate-limit на попытки входа |
| 🔑 **Scoped API keys** | Токены для программного доступа с ровно теми правами, что выданы, не всей учёткой |
| 🖥️ **Живая консоль** | Вывод контейнера и статус online/offline переживают обрыв сокета, сон ноутбука, рестарт демона — с автопереподключением, а не мёртвой вкладкой до ручного F5 |
| 🌐 **Мульти-нода** | Одна панель, любое число нод-демонов на токен-протоколе. Добавить мощность — значит указать `install.sh` на новый сервер |
| 🥚 **Модульные eggs** | Minecraft, Python-приложения, Discord/Telegram-боты, любой Docker-образ — процесс деплоя не знает и не обязан знать, что внутри |
| 🗄️ **Базы данных из коробки** | Регистрируешь MySQL/MariaDB-хост один раз — дальше сервера получают свою БД и учётку по кнопке, без ручных `GRANT` |
| ⏱️ **Реальные расписания** | Cron-задачи на power-действия и команды, атомарный claim, чтобы два тика не выстрелили одной задачей дважды |
| 📁 **Файловый менеджер** | Браузер файлов сервера прямо в панели: правка, загрузка/скачивание, папки, защита от path traversal |
| 🔐 **SFTP из коробки** | Свой публичный ключ в аккаунте — и `sftp username.server-id@node:2022` работает в любом клиенте, без пароля |
| 🌐 **Свои домены** | Привязать домен к серверу прямо в настройках — nginx-реверс-прокси и HTTPS через certbot настраиваются автоматически |
| 💾 **Бэкапы** | Архив всех файлов сервера по кнопке или по расписанию, с исключениями по маске, restore/download/delete прямо из панели |
| 📊 **Активность и аудит** | Единая лента событий (кто, что, когда, с какого IP), доступна только админам |

## 🏗️ Структура репозитория

```
backend/    панель-мастер (Go) — REST API, JWT/2FA/RBAC, WS-шлюз для браузера
daemon/     wingsd — демон на каждой ноде (Go) — Docker, cgroups, консоль
frontend/   SPA (React + Vite + TS), стили — panel.css (собственный дизайн)
website/    маркетинговый сайт (чистый Python + HTML/CSS/JS, деплоится через
            собственный же egg "Python: Website" — догфудинг)
docs/       архитектура, схема БД, протокол panel ↔ wingsd
scripts/    установщик: один файл на этап (docker/database/panel/daemon/...)
```

## 🧪 Локальная разработка

### 1. База данных
```bash
createdb panel
psql panel -f backend/migrations/0001_init.sql
# + остальные backend/migrations/000N_*.sql по порядку
```

### 2. Панель (backend)
```bash
cd backend
go run ./cmd/panel   # :8080, конфиг — через ENV, см. internal/config
```

### 3. Демон (daemon), на каждой ноде
```bash
cd daemon
export WINGSD_DAEMON_TOKEN=<токен, выданный панелью при создании ноды>
go run ./cmd/wingsd  # :8443, требует доступный Docker daemon
```

### 4. Фронтенд
```bash
cd frontend
npm install
npm run dev   # http://localhost:5173, проксирует /api и /ws на :8080
```

`go build ./... && go vet ./...` (оба модуля) и `npm run build`
(`tsc --strict` + vite) проходят чисто — это минимальная планка перед любым PR.
